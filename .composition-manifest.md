## 2026-05-12 re-emission βÄî css-isolation contract

Re-emitted `review-mode.css` and class references in `review-mode.js` against the new `library/features/review-widget/css-isolation.md` atomic (blob `368780bcf2892e7bffe4f7b74896278fbcc028f2`). Three real-world bugs the glinda adoption surfaced βÄî host-page button styles cascading into widget buttons, missing dashed outline on has-comment elements, hover pill staying visible after cursor exit βÄî drove the isolation contract upstream as canonical spec. The original 2026-05-10 composition predated the spec and emitted CSS without the isolation guardrails. This re-emission honors the contract:

- Class prefix changed from `.rv-` to canonical `.review-` (per `css-isolation.md` ¬ß"The contract" item 2). 50+ CSS rules renamed; 80+ JS class references renamed mechanically.
- BEM button variants βÄî `.review-btn--primary`, `.review-btn--secondary`, `.review-btn--danger` βÄî replacing the previous `data-variant` driven styling alone. Two JS button creations updated.
- Chrome reset block added near top of CSS βÄî `all: revert; font-family: ...; box-sizing: border-box; line-height: 1.4;` on every chrome root (banner, sidebar, modal, modal-backdrop, toast, toast-stack, toggle, pill, pill-container, pill-status). Also `box-sizing: border-box` propagated to all descendants of chrome roots.
- Display-none baseline on `.review-pill` made explicit, non-overrideable by host page (per `pill-hover-deepest.md` ¬ß"Required emission"). Hover override uses high-specificity `[data-review-mode="on"] [data-comment-id]:hover:not(:has([data-comment-id]:hover)) > .review-pill-container > .review-pill { display: inline-flex; }`.
- Button isolation block emitted with explicit `font-family: inherit`, `background: transparent`, `color: inherit`, `border: 1px solid transparent`, `padding: var(...)`, `cursor: pointer` (per `components/button.md` ¬ß"Required emission"). Host-page `button { ... }` rules no longer cascade into widget buttons.
- Has-comment outline rule added: `outline: 1px dashed var(--review-accent-warm-low); outline-offset: 2px; position: relative;` on every `[data-comment-id]` element (per `anchor-strategy.md` ¬ß"Visual treatment").
- Container/pill `pointer-events` pair preserved (`pointer-events: none` on `.review-pill-container`, `pointer-events: auto` on `.review-pill`).
- Spotlight class renamed from `.rv-spotlight` to `.review-spotlit` (per `css-isolation.md` ¬ß"Spotlight class" naming).
- Pill host class renamed from `.rv-pill-host` to `.review-pill-container` (per `css-isolation.md` ¬ß"Required emissions" naming).
- Comment row class renamed from `.rv-row` to `.review-comment-row` (per `css-isolation.md` ¬ß"The contract" listing).
- CSS custom properties renamed from `--rv-*` to `--review-*` for consistency (internal scoping; not part of the isolation contract proper, but kept aligned).

Verified against `css-isolation.md` ¬ß"Verifying isolation at composition time" five-step check statically (browser-side verification still required on next deploy):

1. **Inert path.** No `<html data-review-mode="on">` set; CSS rules all gated under that selector; no rule fires when widget is off. β€™ (Static grep: zero unscoped widget rules.)
2. **Active path.** Bootstrap sets `<html data-review-mode="on">` after CSS + module load; chrome renders. β€™ (`review-bootstrap.js` line ~83 sets the attribute.)
3. **Hover pill behavior.** Default `display: none` on `.review-pill`; `:hover:not(:has(...))` override reveals only on deepest hovered anchored element; pill hides on cursor exit because the override stops matching. β€™ (CSS rules present.)
4. **Button isolation.** `.review-btn` carries `font-family: inherit`, `background: transparent`, `color: inherit`, explicit border + padding. Glinda's site-button cascade cannot win against the `[data-review-mode="on"] .review-btn` specificity floor. β€™ (CSS rule emitted in full.)
5. **Has-comment outline.** Every `[data-comment-id]` gets `outline: 1px dashed`; deepens to solid `accent-warm` on `[data-has-comments="true"]`. β€™ (CSS rule emitted; JS sets the `data-has-comments` attribute via `decorateHasComments()` after each RTDB update.)

Browser-side verification still required after the next push lands on the public mirror βÄî load `?review=1`, hover an anchored paragraph, confirm exactly one pill appears, confirm a button in the composer modal looks like the widget variant (purple primary, indigo secondary, red danger) and not Glinda's site-button style.

## HTML script-tag wiring

Each page under `public/method/**/index.html` must include, in the `<head>`, BEFORE any other script:

```html
<!-- on /method/ (depth 0) -->
<script src="./contact-form.config.js"></script>
<script src="./review-bootstrap.js" defer></script>

<!-- on /method/workshops/trial-method/ (depth 2) -->
<script src="../../contact-form.config.js"></script>
<script src="../../review-bootstrap.js" defer></script>
```

The widget self-locates via `document.currentScript`, so deeper nesting picks up siblings correctly. The order matters: `contact-form.config.js` must be loaded BEFORE `review-bootstrap.js` so `window.GLINDA_CONTACT_CONFIG` is populated when the bootstrap checks it.
